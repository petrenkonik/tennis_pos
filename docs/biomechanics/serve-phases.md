# Биомеханика подачи: фазы, события, метрики

> Предметная справка. Это **источник истины** для терминологии и метрик при работе над детекцией фаз и ошибок.
> Основано на 8-стадийной модели Chow et al. (PMC/NIH) и подтверждено систематическими обзорами 2023–2024.

## Макро-структура: 3 фазы → 8 стадий

```
┌─────────────────────────────────────────────────────────────────┐
│                  ПОЛНАЯ ПОДАЧА                                  │
├──────────────────┬──────────────────────┬────────────────────────┤
│  PREPARATION     │    ACCELERATION      │    FOLLOW-THROUGH      │
│  (подготовка)    │    (разгон/удар)     │    (завершение)        │
├──────────────────┼──────────────────────┼────────────────────────┤
│ 1. Stance        │ 5. External rotation │ 8. Follow-through      │
│ 2. Release       │    (Trophy)          │                        │
│ 3. Toss          │ 6. Internal rotation │                        │
│ 4. Knee flexion  │ 7. Contact           │                        │
└──────────────────┴──────────────────────┴────────────────────────┘
```

## Ключевые события (events) — основа разбиения на фазы

Эти события мы детектим из pose trajectory и используем как **границы фаз**.

| Событие | Когда | Как детектить (намёк) | Важность |
|---|---|---|---|
| **Release** | Мяч покидает руку | Тосс-рука раскрывается, начинается восходящая траектория мяча | граница prep→ |
| **Peak toss height** | Верхняя точка подброса | Локальный максимум высоты мяча | метрика Layer 2 |
| **Max knee flexion** | Макс. сгибание коленей | Минимум «высоты бёдер» / max угол сгиба | метрика + часть trophy |
| **Trophy position** | «Трофей»: ракетка за головой, тосс-рука вытянута, колени согнуты | Комбинация: ракетка выше головы + тосс-рука вверх + колени согнуты | **ключевая фаза**, частая проблема у любителей |
| **Contact** | Удар по мячу | Мяч встречается с ракеткой; макс. вытяжение вверх; резкая смена траектории мяча | **граница accel→**, ключевой момент |
| **Follow-through start** | После контакта | Ракетка идёт вниз/вперерез туловища | граница →follow-through |

> ⚠️ **Для прототипа** приоритет — детектировать **release, trophy, contact, follow-through**. Этого достаточно для осмысленного 4-фазного разбиения. Более мелкие стадии (stance, knee flexion отдельно) — Layer 2/future.

## Метрики по фазам

### Preparation
- **Toss height** — высота подброса (от release до peak). Слишком низкий → спешка; слишком высокий → потеря тайминга.
- **Toss consistency** — вариативность подброса между подачами (future, для серии).
- **Toss placement** — куда упадёт мяч относительно тела (front/side).

### Acceleration
- **Trophy position quality** — ракетка за головой вертикально? тосс-рука вытянута? колени согнуты?
- **Knee bend angle** — угол сгиба коленей в trophy (типично 20–35° у intermediate).
- **Kinetic chain** — последовательность: ноги → бёдра → туловище → плечо → рука. (Сложно для CV 2D — частично future.)
- **Contact point height** — высота контакта относительно вытянутой руки.
- **Contact point placement** — положение контакта относительно тела (впереди/над/сзади).

### Follow-through
- **Ракетка завершает движение** на противоположной стороне тела (для правшей — слева).
- **Баланс** — не падает ли игрок после подачи.

## Типичные ошибки любителей 3.0–4.0

Эти ошибки — кандидаты на rule-based правила (см. `skills/serve-error-detection/`).

### Trophy position
1. **«Ловит» ракетку в trophy** — слишком долго стоит в trophy, нет плавного перехода в acceleration.
2. **Ракетка не опускается за голову** (нет racket drop) — теряет энергию разгона.
3. **Тосс-рука падает слишком рано** — теряет «указатель» на мяч.

### Toss
4. **Подброс слишком далеко назад** — вынужден отклоняться назад при контакте.
5. **Подброс слишком низкий** — не успевает сделать полный замах.
6. **Непоследовательный подброс** — мяч летит в разные места (future, серия подач).

### Contact
7. **Контакт слишком низко** — бьёт сгибом локтя вместо вытянутой руки.
8. **Контакт позади тела** — бьёт назад, мяч уходит в сетку/длинный.
9. **Слишком мало сгиба коленей** — не использует ноги для энергии.

### Follow-through
10. **Резкая остановка** после удара — нет follow-through, риск травмы плеча.
11. **Потеря баланса** — падает после подачи.

## Источники порогов

> ⚠️ Конкретные числовые пороги (углы, таймстампы) **не фиксированы здесь** — они определяются эмпирически на тестовых данных и фиксируются в `src/constants/biomechanics.ts` с комментарием-источником.

Возможные источники:
- Chow et al. (2012) — 8-stage модель, общие диапазоны
- MDPI 2024 / Frontiers 2024 — kinematic ranges для разных уровней игроков
- OnCourtAI — практические метрики из AI-анализа
- Эмпирическая калибровка на тестовых подачах (preferred для прототипа)

## Что НЕ входит в прототип

- **3D кинематика.** MediaPipe даёт 3D landmarks, но надёжность depth-оценки варьируется. На прототипе полагаемся преимущественно на 2D + аккуратное использование visibility-метрик.
- **Скорость подачи.** Требует трекинга мяча (YOLO) и калибровки перспективы — отложено.
- **Сравнение с эталоном (Layer 3)** — частично в прототипе, но полная overlay-модель требует набора эталонных видео.
- **Серия подач / прогресс** — future, прототип работает с одной подачей за раз.
