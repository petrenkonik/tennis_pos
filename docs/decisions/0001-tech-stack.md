# ADR-0001: Технологический стек прототипа

**Дата:** 2026-06-20
**Статус:** Принято

## Контекст

Нужно выбрать платформу, источник видео и CV-стек для прототипа приложения анализа подачи.

## Рассмотренные варианты

### Платформа
- iOS нативно (Swift, Core ML)
- Android нативно (Kotlin, ML Kit)
- Кросс-платформа (RN/Flutter)
- **Веб-прототип (валидация CV)** ✅

### Источник видео
- Live streaming с камеры
- **Загрузка видео-файла** ✅

### CV-стек
- Python backend + MediaPipe/YOLO
- **Всё в браузере (MediaPipe JS / TF.js)** ✅
- Notebook/Streamlit

## Решение

**Веб-прототип, всё в браузере через MediaPipe JS / TF.js, загрузка видео-файлов.**

## Обоснование

1. **Веб-прототип** позволяет быстро итерировать по CV-алгоритмам без боли мобильной интеграции, сборки, деплоя на устройство. Цель прототипа — валидировать *алгоритмы*, а не UX.
2. **Всё в браузере** — zero серверной инфраструктуры. Соответствует будущему мобильному продукту (on-device обработка), упрощает деплой (статика), устраняет вопросы приватности (видео не покидает устройство).
3. **Загрузка видео-файла** — пользователь контролирует условия съёмки, нет real-time ограничений браузера, нет проблем с тем что устройство нужно держать неподвижно. Проще отлаживать на известных тестовых видео.

## Последствия

### Положительные
- Нет бэкенда → нет затрат на хостинг, нет вопросов приватности
- Деплой = статический хостинг (GitHub Pages / любой CDN)
- Приближен к финальному мобильному UX (on-device)
- Быстрая итерация на алгоритмах

### Отрицательные / риски
- MediaPipe JS может иметь **меньший FPS** чем нативный мобильный SDK — приемлемо для анализа видео-файла (не real-time)
- Меньше контроля над детализацией моделей чем в Python-пайплайне — компенсируем тщательной обработкой landmarks
- Браузерные ограничения по памяти для длинных видео — для прототипа ограничим длину (напр. ≤30с одной подачи)

### Ограничения, которые это накладывает
- **Никаких серверных вызовов, API-ключей, бэкенда.** Загрузка MediaPipe model weights из CDN — допустима.
- ML-классификаторы и LLM (future-work) потребуют пересмотра этого ADR когда до них дойдёт дело.
