# tennis_pos

Веб-прототип приложения для анализа подачи в теннисе с Computer Vision и pose tracking.

## Что это

Пользователь-любитель (уровень игры 3.0–4.0) загружает видео своей подачи. Приложение:

1. **Распознаёт позу** через MediaPipe Pose (33 точки скелета, покадрово)
2. **Разбивает подачу на фазы** по ключевым событиям (preparation → trophy → contact → follow-through)
3. **Находит ошибки в движении** через rule-based правила биомеханики
4. **Показывает фидбек** на трёх уровнях глубины — от простых советов до сравнения скелета с эталоном профи

Цель прототипа — валидировать, что CV-пайплайн реально умеет разбивать подачу на фазы и находить ошибки.

## Стек

- **Платформа:** веб, всё работает в браузере
- **Pose tracking:** MediaPipe Pose (JS / TF.js), on-device
- **Анализ:** rule-based, без серверной инфраструктуры
- **Источник видео:** загрузка файла (не real-time)

### Ручная проверка сквозного потока (критерий успеха MVP)
1. `npm run dev`, открыть приложение.
2. Выбрать клип подачи (сбоку, игрок целиком в кадре, ≤30с) и handedness.
3. Дождаться прогресс-бара обработки.
4. Убедиться, что:
   - на видео рисуется скелет и подпись текущей фазы;
   - полоса фаз показывает 4 сегмента (Подготовка/Трофей/Разгон/Завершение);
   - показан ≥1 совет либо «Ошибок не найдено».
